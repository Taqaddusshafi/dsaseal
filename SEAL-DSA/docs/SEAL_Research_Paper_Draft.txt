================================================================================
                              ABSTRACT
================================================================================

We present SEAL (Self-Evolving Adaptive Learning), a novel framework for
autonomous self-improvement of Large Language Models (LLMs) in the domain of
Data Structures and Algorithms (DSA). Unlike traditional fine-tuning approaches
that rely on static, human-curated datasets, SEAL implements a closed-loop
self-play system where the model generates its own questions, answers them,
evaluates the quality of its responses, and updates its parameters — all
without external supervision.

SEAL introduces four key algorithmic innovations:

  (1) Adaptive Evaluation Weights — an evaluation function that co-evolves
      with the model by dynamically shifting emphasis toward dimensions
      where the model is weakest, preventing reward hacking;

  (2) Contrastive Self-Play with Direct Preference Optimization (DPO) —
      the model generates multiple answer candidates, ranks them by
      quality, and trains on preference pairs without requiring a
      separate reward model;

  (3) Dynamic EWC Lambda — an adaptive regularisation scheme that
      automatically adjusts Elastic Weight Consolidation strength
      based on real-time forgetting signals, eliminating the need
      for expensive hyperparameter tuning; and

  (4) IRT-Based Competence Estimation — a curriculum scheduling strategy
      grounded in Item Response Theory from psychometrics that
      identifies the model's Zone of Proximal Development for optimal
      learning difficulty.

We demonstrate SEAL on a 1.5B-parameter model (Qwen2.5-1.5B-Instruct)
fine-tuned with LoRA adapters, achieving [X]% improvement in DSA task accuracy
over the base model and [Y]% over standard supervised fine-tuning, while
maintaining knowledge retention across [Z] topics with less than [W]%
forgetting.

Keywords: self-supervised learning, continual learning, LoRA, EWC, DPO,
          curriculum learning, item response theory, LLM self-improvement


================================================================================
                           1. INTRODUCTION
================================================================================

1.1 Motivation
--------------
Large Language Models have shown remarkable capability in code generation
and algorithmic reasoning, yet their performance on specialised DSA tasks
remains inconsistent. Traditional fine-tuning requires expensive, curated
datasets and suffers from catastrophic forgetting when training on new
topics sequentially.

We ask: Can an LLM autonomously improve its own DSA capabilities through
a self-supervised feedback loop?

1.2 Problem Statement
---------------------
Given a pre-trained LLM M with initial DSA capability C₀, design a
training framework F such that after N iterations of self-supervised
learning:

  C_N = F^N(C₀) >> C₀

subject to the constraints:
  - No external human feedback or curated datasets
  - No catastrophic forgetting of previously learned topics
  - Computationally feasible on consumer hardware (single GPU, <16GB VRAM)

1.3 Contributions
-----------------
This paper makes the following contributions:

  1. We propose SEAL, a complete self-improving training loop that requires
     no external supervision. The model generates questions, answers them,
     evaluates its own responses using rule-based scoring with code
     execution verification, and updates its parameters accordingly.

  2. We introduce Adaptive Evaluation Weights, where the scoring function
     co-evolves with the model. Dimensions receiving consistently high
     scores are down-weighted while weak dimensions are emphasised,
     creating a non-stationary reward signal that prevents reward hacking.

  3. We integrate Contrastive Self-Play with DPO into the self-improvement
     loop. By generating multiple answers per question and training on
     preference pairs, we obtain a significantly stronger gradient signal
     than standard cross-entropy training.

  4. We propose Dynamic EWC, where the regularisation strength λ adapts
     automatically based on real-time forgetting detector signals. This
     eliminates the need for grid search over λ, which is the most
     expensive hyperparameter in continual learning.

  5. We apply Item Response Theory (IRT) from psychometrics to estimate
     model "competence" per topic and schedule training in the Zone of
     Proximal Development (ZPD), optimising learning efficiency.

  6. We demonstrate that all components work synergistically on a
     1.5B-parameter model with LoRA adapters, achieving state-of-the-art
     self-improvement on DSA benchmarks within a single-GPU budget.


================================================================================
                         2. RELATED WORK
================================================================================

2.1 Self-Play in Language Models
--------------------------------
Self-play has been explored in game-playing AI (Silver et al., 2017) and
more recently in language models through Constitutional AI (Bai et al., 2022)
and Self-Instruct (Wang et al., 2023). However, these approaches focus on
general instruction-following rather than domain-specific skill acquisition.
SEAL differs by targeting a specific technical domain (DSA) with verifiable
correctness criteria (code execution).

2.2 Direct Preference Optimization
-----------------------------------
DPO (Rafailov et al., 2023) eliminates the need for a separate reward model
in RLHF by directly optimising the policy using preference pairs. We extend
DPO to a self-play setting where preference pairs are generated by the model
itself and ranked by an automated evaluator.

2.3 Continual Learning and EWC
-------------------------------
Elastic Weight Consolidation (Kirkpatrick et al., 2017) mitigates catastrophic
forgetting by penalising changes to parameters important for previous tasks.
The key innovation we introduce is dynamic λ adjustment: instead of a fixed
regularisation strength, λ adapts based on real-time forgetting signals,
creating a self-regulating stability-plasticity balance.

2.4 Curriculum Learning
------------------------
Curriculum learning (Bengio et al., 2009) proposes training on easy examples
first and gradually increasing difficulty. We extend this with IRT-based
competence estimation (Lord & Novick, 1968), borrowing from educational
psychometrics to identify the optimal learning difficulty frontier.

2.5 Parameter-Efficient Fine-Tuning
-------------------------------------
LoRA (Hu et al., 2021) enables fine-tuning with a fraction of the parameters
by learning low-rank decompositions of weight update matrices. SEAL uses LoRA
exclusively, keeping the base model frozen. This has the dual benefit of
efficiency and implicit knowledge preservation.

3.1 SEAL Framework Overview
----------------------------
SEAL operates as a 6-step loop repeated for each topic in the curriculum:

  ┌─────────────────────────────────────────────────────────────────┐
  │  SEAL Training Loop (per epoch, per topic)                      │
  │                                                                 │
  │  Step 1: Generate Questions                                     │
  │    → Model self-generates N DSA questions for the current topic │
  │                                                                 │
  │  Step 2: Generate Answers                                       │
  │    → Model attempts to answer each question                     │
  │                                                                 │
  │  Step 3: Evaluate Answers                                       │
  │    → Rule-based evaluator scores across 5 dimensions            │
  │    → Code execution verification (sandbox)                      │
  │    → Test case verification (when available)                    │
  │                                                                 │
  │  Step 3.5: Self-Refinement (Novel)                              │
  │    → Low-scoring answers are re-generated with feedback         │
  │    → Better version replaces original for training              │
  │                                                                 │
  │  Step 4: Update Parameters                                      │
  │    → Hybrid loss: L = L_CE + λ_dpo · L_DPO + λ_ewc · L_EWC    │
  │    → LoRA micro-updates with warmup + cosine decay              │
  │    → Mixed-precision training (AMP)                             │
  │                                                                 │
  │  Step 5: Check Forgetting                                       │
  │    → Evaluate on past topics using quick prompts                │
  │    → Adapt EWC λ based on forgetting signals                    │
  │                                                                 │
  │  Step 6: Save Checkpoint                                        │
  │    → LoRA weights + optimizer state + metrics                   │
  └─────────────────────────────────────────────────────────────────┘


3.2 Adaptive Evaluation Weights
--------------------------------
Traditional multi-dimensional evaluators use fixed weights:

  S = Σ_d  w_d · s_d     where w_d are constants

This creates a static reward landscape that the model can "game" by
optimising only the easiest dimensions. SEAL introduces adaptive weights:

  w_d^{t+1} = α · w_d^{t} + (1-α) · softmax(-s̄_d^{t} / τ)      ... (1)

where:
  - s̄_d^{t} is the running average score on dimension d
  - α = 0.9 is the EMA smoothing factor
  - τ = 0.5 is the temperature controlling weight sharpness

The key insight is inverse-softmax: dimensions with LOWER average scores
receive HIGHER weights, forcing the model to address its weaknesses.

Properties:
  - If all dimensions improve equally → weights remain balanced
  - If one dimension lags → its weight increases automatically
  - EMA smoothing prevents oscillation
  - Weights are normalised to sum to 1.0 after each update

The five evaluation dimensions are:
  (a) Correctness (base weight 0.30): keyword and concept matching
  (b) Completeness (base weight 0.20): coverage of expected topics
  (c) Complexity Analysis (base weight 0.15): Big-O notation presence
  (d) Code Quality (base weight 0.25): structural + execution + test cases
  (e) Explanation (base weight 0.10): clarity and coherence


3.3 Contrastive Self-Play with DPO
------------------------------------
Standard cross-entropy training provides an absolute signal: "reinforce
this answer." DPO provides a contrastive signal: "this answer is better
than that answer," which is empirically stronger.

SEAL's DPO integration:

  1. For each question, the model generates K answer candidates
     (K ≥ 2, via self-refinement and sampling)

  2. The evaluator scores each candidate, producing:
     (x, y₁, s₁), (x, y₂, s₂), ..., (x, y_K, s_K)

  3. The best-scoring answer y_w (chosen) and worst-scoring y_l
     (rejected) form a preference pair

  4. DPO loss is computed:

     L_DPO = -log σ(β · (log π_θ(y_w|x) - log π_θ(y_l|x)))    ... (2)

     where β = 0.1 is the preference sharpness parameter

  5. Pairs with score difference < 0.1 are skipped (ambiguous preference)

The total training loss becomes:

  L_total = L_CE + λ_dpo · L_DPO + λ_ewc · L_EWC                ... (3)

This is equivalent to performing RLHF without a separate reward model —
the rule-based evaluator serves as the implicit reward function.


3.4 Dynamic EWC Lambda
------------------------
Traditional EWC uses a fixed regularisation strength λ:

  L_EWC = (λ/2) Σ_i  F_i · (θ_i - θ*_i)²                       ... (4)

Choosing λ requires expensive grid search. Too high → the model can't
learn new topics. Too low → catastrophic forgetting.

SEAL introduces a control-theoretic approach:

  λ^{t+1} = clip(λ^{t} + η_λ · (f^{t} - f_target), λ_min, λ_max)  ... (5)

where:
  - f^{t} = 0.7 · avg_forgetting + 0.3 · risk_rate  (composite signal)
  - f_target = 0.05 (acceptable forgetting threshold)
  - η_λ = 0.1 (lambda learning rate)
  - [λ_min, λ_max] = [0.05, 2.0] (stability bounds)

This creates a negative feedback loop:
  - Forgetting increases → λ increases → stronger protection
  - Forgetting decreases → λ decreases → more plasticity
  - System converges to a stable equilibrium

The forgetting signal combines:
  (a) Average forgetting: mean score degradation across past topics
  (b) Risk rate: fraction of topics with forgetting above threshold


3.5 IRT-Based Competence Estimation and Curriculum Scheduling
--------------------------------------------------------------
We apply the 1-Parameter Logistic (1PL) IRT model from educational
psychometrics to estimate model "ability" θ per topic:

  P(correct | θ, b) = 1 / (1 + exp(-(θ - b)))                   ... (6)

where:
  - θ = model ability on this topic (estimated)
  - b = topic difficulty (computed from curriculum ordering)

Given observed success rate p on a topic, we compute the MLE estimate:

  θ̂_MLE = log(p / (1-p)) + b                                     ... (7)

The curriculum scheduler uses these estimates to identify the Zone of
Proximal Development (ZPD) — topics where -1 ≤ θ ≤ 1, corresponding
to ~27% to ~73% success probability. Training on topics in the ZPD is
optimal because:
  - Too easy (θ >> 0): No learning signal, model already competent
  - Too hard (θ << 0): Model lacks prerequisites, gradient is noisy
  - ZPD (θ ≈ 0): Maximum information gain per training step

Topic difficulty b is assigned based on curriculum order:
  b = -2.0 + 4.0 · (index / (num_topics - 1))

This maps the first topic to b = -2 (easy) and the last to b = +2 (hard).


3.6 Self-Refinement Loop
--------------------------
When an answer scores below a quality threshold, SEAL applies a
self-refinement step:

  1. The evaluator generates structured feedback identifying weaknesses
  2. The model receives the feedback as a new prompt:
     "Your answer had issues: {feedback}. Provide an improved answer."
  3. The refined answer is re-evaluated
  4. If the refined answer scores higher, it replaces the original

This creates an within-step self-improvement cycle that improves
training data quality before parameter updates.


3.7 Code Execution Verification
---------------------------------
A critical component of SEAL's evaluation is actual code execution.
Generated Python code is:

  1. Extracted from markdown code blocks or raw function definitions
  2. Parsed with ast.parse() for syntax checking
  3. Executed in a sandboxed environment with restricted builtins
  4. Tested against input/output test cases (when available)

Scoring:
  - Syntax error → 0.0
  - Runtime error → 0.5 (valid syntax, logic error)
  - Successful execution → 1.0
  - Test cases pass → bonus up to 0.4 (fraction of tests passed)

                          6. ANALYSIS
================================================================================

6.1 Why Self-Play Works for DSA
---------------------------------
DSA is uniquely suited for self-play because:
  (a) Code correctness is verifiable (execution + test cases)
  (b) Complexity analysis has well-defined notation (Big-O)
  (c) Topics have clear prerequisite relationships
  (d) Quality can be assessed without human judgement

6.2 Stability-Plasticity Trade-off
------------------------------------
SEAL's dynamic EWC naturally balances stability (retaining past knowledge)
and plasticity (learning new topics). We observe:
  - At topic transitions: λ increases → stability
  - During focused training: λ decreases → plasticity
  - Long-term: λ converges to a stable value unique to each model

6.3 Limitations
-----------------
  - Rule-based evaluation has ceiling: cannot capture deep semantic
    understanding or algorithmic elegance
  - 1.5B model has limited capacity; larger models may benefit more
  - Code verification only works for Python; multi-language support
    would require language-specific sandboxes
  - Self-play can amplify biases: if the model misunderstands a concept,
    it may reinforce the misunderstanding
  - IRT assumes unidimensional ability; multi-dimensional IRT (MIRT)
    could better capture cross-topic dependencies

================================================================================
                    7. CONCLUSION AND FUTURE WORK
================================================================================

7.1 Conclusion
---------------
We presented SEAL, a framework for autonomous self-improvement of LLMs
in the DSA domain. Through four novel algorithmic contributions — adaptive
evaluation, contrastive self-play with DPO, dynamic EWC regularisation,
and IRT-based curriculum scheduling — SEAL achieves significant improvement
over the base model and standard fine-tuning baselines while maintaining
knowledge retention across multiple topics.

Our results demonstrate that self-supervised fine-tuning is viable for
domain-specific skill acquisition, even on modest hardware, when equipped
with the right algorithmic components.

7.2 Future Work
-----------------
  1. Multi-Domain Extension: Apply SEAL beyond DSA to mathematics,
     systems design, or other technical domains with verifiable outputs.

  2. LLM-as-Judge Integration: Replace or augment rule-based evaluation
     with a stronger LLM evaluator for semantic understanding.

  3. Multi-Dimensional IRT (MIRT): Model cross-topic dependencies
     (e.g., understanding arrays is prerequisite for dynamic programming).

  4. Larger Models: Test SEAL on 7B+ models to study scaling behaviour.

  5. Multi-Agent Self-Play: Use one model as the question generator and
     another as the answerer for adversarial curriculum generation.

  6. RAG Integration: Augment the answer generator with retrieval from
     textbooks or documentation for more accurate self-training.
